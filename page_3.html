Selecteren van elementen

De elementen in de DOM kan je ook selecteren. Hiervoor heb je al gelezen dat er een aantal methodes zijn. Je kan een getElements* versie gebruiken of een querySelector/ querySelectorAll.

De verschillen

querySelector en getElementById geven, mits deze bestaat, altijd 1 element terug. Je moet wel altijd even controleren of die bestaat/ gevuld is. Dit kan met een simpel if-statement 

                    
                        let myElement = document.querySelector('.start');
                        if (myElement) {
                            // Je weet in dit deel dat je variabel bestaat/ gevuld is.
                        }

querySelectorAll en getElementsBy* geven altijd een collectie terug. Dit is een lijst/ Array waar je met een foreach/ for of while loop door heen moet/ kan lopen. Je hoeft niet te controleren of de collectie bestaat, het systeem kan ook een lege collectie terug sturen, je kan er altijd door heen lopen. 

                    
                        let myElements = document.querySelectorAll('.normal_text');
                        for (let i = 0; i < myElements.length; i++) {
                            console.log(myElements[i]);
                        }

Wat heeft de voorkeur?

In modern Javascript en in moderne frameworks wordt eigenlijk altijd de voorkeur gegeven aan het selecteren met de querySelector en querySelectorAll methode. En daarnaast bij voorkeur ook altijd op class-name, en niet op ID.

Weg met ID

Sterker nog, het gebruiken van een ID op een element in HTML doen we eigenlijk niet meer. Er zijn betere alternatieven. Veel developers gebruiken een unieke class-name of gebruiken het data-* attribute om informatie op een element te stoppen.

Classes all the way!

In het verlengde van het niet werken met een ID werken we ook niet direct op/ met elementen. Dus gebruik ook geen selector op een <button> element (of h1, h2, h3, body, input etc.etc.). Eigenlijk doen we dit in HTML en in CSS. Dus ook het werken in stylesheets doe je primair op classname basis.
